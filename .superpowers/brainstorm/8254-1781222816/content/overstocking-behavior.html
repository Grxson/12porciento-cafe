Stock Validation — Handle Overstocking How?

User tries to add 10 units, but only 5 available. What should happen?

A

Silently Cap at Available Stock

Disable + button when qty reaches max. Qty stepper shows "5 / 5".

Cantidad:
[−] 5 [+]   5 disponibles (máximo)

UX: No error state. User discovers limit naturally. Silent failure mode.

B

Toast Error + Auto-Correct

User clicks +, gets toast: "⚠️ Stock insuficiente. Limitado a 5 unidades." Qty auto-adjusts to 5.

⚠️ Stock insuficiente
Limitado a 5 unidades · Últimas disponibles

UX: Explicit feedback. User knows why limit exists. Educates about urgency.

C

Reject Add, Show Modal

When user tries to add 10 but only 5 exist, modal pops: "¿Deseas proceder con 5 unidades disponibles?"

Stock Limitado

Solo quedan 5 unidades de este producto.

UX: Explicit consent. Prevents accidental partial adds. More friction.
